# Data Processing Agreement (DPA)

**Last Updated:** {{ last_updated }}

## 1. Definitions

- **Data Controller:** {{ company_name }}
- **Data Processor:** Service provider processing data on behalf of the Data Controller
- **Personal Data:** Any information relating to an identified or identifiable natural person
- **Processing:** Any operation performed on personal data (collection, storage, use, etc.)

## 2. Scope and Purpose

This Data Processing Agreement (DPA) governs the processing of personal data by the Data Processor on behalf of the Data Controller in accordance with the General Data Protection Regulation (GDPR/AVG).

## 3. Processing Activities

### 3.1 Categories of Data Subjects

- Customers and users of the service
- Employees and contractors
- Business partners

### 3.2 Types of Personal Data

- Contact information (name, email, address, phone)
- Account credentials
- Payment information
- Usage data and analytics
- Communication records

### 3.3 Processing Purposes

- Service delivery and support
- Payment processing
- Communication and notifications
- Analytics and service improvement
- Legal compliance

## 4. Data Processor Obligations

The Data Processor agrees to:

- Process personal data only in accordance with documented instructions from the Data Controller
- Ensure confidentiality of personal data
- Implement appropriate technical and organizational measures to ensure security
- Assist the Data Controller in responding to data subject requests
- Notify the Data Controller without undue delay of any data breach
- Maintain records of processing activities
- Return or delete personal data upon termination of services

## 5. Data Controller Obligations

The Data Controller agrees to:

- Provide clear instructions for data processing
- Ensure lawful basis for processing
- Respond to data subject requests
- Ensure data accuracy and completeness

## 6. Security Measures

The Data Processor implements the following security measures:

- Encryption of data in transit and at rest
- Access controls and authentication
- Regular security assessments
- Incident response procedures
- Staff training on data protection

## 7. Sub-Processors

The Data Processor may engage sub-processors with prior written consent from the Data Controller. Sub-processors are bound by the same data protection obligations.

## 8. Data Subject Rights

The Data Processor will assist the Data Controller in fulfilling data subject rights, including:

- Right of access
- Right to rectification
- Right to erasure
- Right to restriction of processing
- Right to data portability
- Right to object

## 9. Data Breach Notification

In the event of a personal data breach, the Data Processor will:

- Notify the Data Controller without undue delay (within 72 hours where feasible)
- Provide detailed information about the breach
- Assist in notifying supervisory authorities and data subjects if required

## 10. Audit Rights

The Data Controller has the right to audit the Data Processor's compliance with this DPA, subject to reasonable notice and confidentiality obligations.

## 11. International Transfers

Any transfer of personal data outside the European Economic Area (EEA) will be conducted in accordance with GDPR requirements, including appropriate safeguards such as Standard Contractual Clauses.

## 12. Term and Termination

This DPA remains in effect for the duration of the service agreement. Upon termination, the Data Processor will return or delete all personal data in accordance with the Data Controller's instructions.

## 13. Governing Law

This DPA is governed by the laws of the Netherlands and is subject to the jurisdiction of Dutch courts.
